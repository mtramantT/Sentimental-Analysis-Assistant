Databinding

In angular, databinding is the act of passing data from the back to the front or vice versa. note that the "view" is the part of the component that actually displays information to the user
There are 3 ways to perform one-way databinding. These are techniques where either the data is passed from the front to the back or the back to the front

String Interpolation

In order to perform interpolation, you need to write the item to be evaluated in between double curly braces: {{expression}}. It is essentially a block of typescript as innerHTML.

Examples


2 + 5 * 3 - 1 * 88 = {{2 + 5 * 3 - 1 * 88}}

Property binding is a "one-way binding" technique that binds from the back to the front. We do this by surrounding what would be the attribute of the elemnt with square brackets [like this]

Event Binding

Event binding is the third type of one-way binding. It is done within the element's tag, agian like property binding, except we put it within paranthesis (). We will typically end up wrapping these around actual events as well. Ex: (click)

Two-Way databinding

We can set up two way databinding, where data we input is fed back to a component and can be brought back all in the same action. In order to do this, we invoke both property and event binding. EX: [(ngModel)]
